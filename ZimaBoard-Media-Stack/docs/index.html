ZimaBoard Media Stack

Request → download → organize → stream for movies and TV, plus a books/reading lane — documented with placeholders only.

Overview

This stack covers movies/TV (Jellyfin, Jellyseerr, Sonarr, Radarr, Prowlarr, qBittorrent) and books (Kavita, librarr, Calibre-Web Automated). Hypervisor layout and NAS shares live in companion repos; this page describes the app pipeline and how pieces connect.

LaneJob
WatchJellyfin streams libraries from shared media storage
RequestJellyseerr (or similar) is the human-facing ask UI
AutomateSonarr / Radarr + Prowlarr find and grab; qBittorrent downloads
ReadKavita (+ librarr / Calibre-Web Automated) for books and comics

Storage

Mount media by LABEL or /dev/disk/by-id/... — never by unstable /dev/sdX letters. Player and automation guests should see the same absolute path so hardlinks work.

DataExample pathConsumers
Movies / TV / downloads/data/mediaSonarr, Radarr, qBittorrent, Jellyfin
Books / textbooks/data/books (own disk preferred)Kavita, Calibre-Web Automated, librarr

Keep books off the movies/TV volume unless you intentionally want one big library disk.

Create

Condensed bring-up — adjust names and paths to match your host.

  1. Prepare a guest (LXC with nesting / Docker, or VM) with LAN access and enough disk for configs.
  2. Bind-mount the shared media volume at a fixed path (example /data/media).
  3. Deploy Jellyfin, Jellyseerr, Sonarr, Radarr, Prowlarr, and qBittorrent (compose or preferred install method).
  4. Wire Prowlarr → Sonarr/Radarr; point download client to qBittorrent; point Jellyseerr at Sonarr/Radarr and Jellyfin.
  5. On the books guest/disk, deploy Kavita, Calibre-Web Automated, and optional librarr.
  6. Create libraries in Jellyfin / Kavita pointing at organized folders — not the incomplete downloads root.
  7. Store API keys and passwords in a password manager — never in git.

Use

Day-to-day: ask in Jellyseerr, wait for automation, watch in Jellyfin. Touch *arr and Prowlarr mainly when something fails.

  • Request → Jellyseerr
  • Watch → Jellyfin
  • Stuck download? → qBittorrent, then Sonarr/Radarr activity
  • Missing indexer? → Prowlarr
  • Read → Kavita (library fed by Calibre-Web Automated / librarr)

Safety

  • No passwords, API keys, personal usernames, or private LAN addresses in public documentation.
  • Use placeholders (<JELLYFIN_HOST>, <REQUEST_HOST>, etc.) and set real values only on your infrastructure.
  • Prefer LABEL/by-id disk mounts for removable media.
  • Keep books storage separate when you want media-drive isolation.
  • Respect copyright and local law; this documentation describes software roles only.
  • See SECURITY.md in the repo root for the redaction checklist when present.
